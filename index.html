ParkNav Help Guide

XAVIER UNIVERSITY · ATENEO DE CAGAYAN · ADMIN DASHBOARD

🔌 Arduino Connection
1
Click Connect Arduino in the top-right corner of the header to open the browser's serial port selector.
2
Select the correct COM port your Arduino is plugged into (usually labelled USB Serial or CH340). The system connects at 9600 baud.
3
Once connected, the status pill turns ● green. Parking data will start flowing automatically from the Arduino sensors.
4
To disconnect, click the button again. Note: Web Serial only works in Chrome or Edge browsers.
🅿️ Parking Dashboard Cards
✦
Each card represents one parking lot: Museum (cap. 6), CH (cap. 4), and Loyola (cap. 6).
✦
The large number shows Occupied / Capacity. The gauge bar fills as the lot fills up.
✦
Status chips: ● AVAILABLE = slots free · ● FILLING UP = ≥75% full · ● FULL = at capacity.
✎ Manual Count Adjustment (±)
+
Use the + button on a card or the Campus Total summary box to increase the occupied count by 1 — e.g., a car entered but the sensor missed it.
−
Use the − button to decrease the occupied count by 1 — e.g., a car left without triggering the sensor.
✦
Every adjustment is immediately sent to Google Sheets with a [MANUAL ADJ] tag so you have a full audit trail.
⚠ Adjustments clamp within 0 and the lot's maximum capacity — they cannot go negative or exceed the limit.
🤖 Arduino Simulator
▶
Start Auto — begins a realistic campus traffic simulation with vehicle arrivals, parking, and drop-offs following real time-of-day patterns. Use the Speed slider to control simulation pace.
■
Stop — halts the automatic simulation. The countdown bar resets.
↯
Fire Once — sends a single random update immediately without starting the full auto cycle.
✦
Manual fire buttons (Museum / CH / Loyola / Campus Total) let you trigger a specific lot's update on demand.
The simulator feeds data through the exact same parsing pipeline as the real Arduino — output is identical.
📋 Message Log & Google Sheets
✦
Every parking update is automatically saved to the connected Google Sheet via Apps Script. The log refreshes every 30 seconds.
✦
Use the Search box to filter log entries by keyword. Use the filter buttons (All / Museum / CH / Loyola / Campus Total) to narrow by area.
✦
Click ↻ Refresh Log to manually pull the latest data from Google Sheets at any time.
✦
Log columns: Message · Date · Time · Area · Occupied · Capacity · Free · Status · Fill %.
⚡ Serial Monitor
✦
The serial monitor panel shows every raw message received from the Arduino (or simulator) in real time, color-coded by type: info · warn · err.
✦
Click the panel header to collapse or expand the monitor. It keeps the last 200 lines.
🎨 Theme & Display
⚙
Click the ⚙ gear button (floating lower-right, above the ? button) to toggle between Dark Mode (default) and Light Mode.
?
Click the ? help button (floating lower-right) to show this guide at any time.
